¿What means being a Programmer?

Being a programmer is more than just write code you need more skills than code writing to be a succesfull programer, team work, patience and constant improvement are some of the habilities that programmers need in order to face daily challenges

¿What is a Program?

A program is a Software entity that allows us to solve a specific problem. We say that it is a Software entity since it is not a set of physical components (such as hardware) but allows us to manipulate them. When we talk about a program, we refer to a set of logical elements (called Algorithms) that are specified, designed and later implemented by the development team.

Common uses of a program:

  • Solve the accounting of a company.
  • Manipulate a robot.
  • Read and play audio, video, text files, etc.
  • Perform mathematical operations.

¿What is an Algorithm?

In formal terms, an algorithm is a fundamental logical unit. That is, a portion of reasoning. An algorithm allows you to solve a problem with a specific objective. We usually make an analogy between algorithms and cooking recipes. Thats it, if our problem to solve is 'Cooking a vanilla sponge cake', we can propose our recipe in an orderly way like an algorithm to solve the recipe or "the problem"


unlike programs, they solve more general problems. Among these more general problems we can include:

  • Sort a messy data set.
  • Calculate the verification digit of the ID.
  • Encryption of a password.
  • Calculate the area of ​​a polygon.

¿What is a Flowchart?

A flowchart is simply a graphical representation of steps. It shows steps in sequential order and is widely used in presenting the flow of algorithms, workflow or processes. Typically, a flowchart shows the steps as boxes of various kinds, and their order by connecting them with arrows.

¿What is PseudoCode?

Pseudocode is an informal way of programming description that does not require any strict programming language syntax or underlying technology considerations. It is used for creating an outline or a rough draft of a program. Pseudocode summarizes a program’s flow, but excludes underlying details. System designers write pseudocode to ensure that programmers understand a software project's requirements and align code accordingly.
