¿What means being a Programmer?

Being a programmer is more than just write code you need more skills than code writing to be a succesfull programer, team work, patience and constant improvement are some of the habilities that programmers need in order to face daily challenges

¿What is a Program?

A program is a Software entity that allows us to solve a specific problem. We say that it is a Software entity since it is not a set of physical components (such as hardware) but allows us to manipulate them. When we talk about a program, we refer to a set of logical elements (called Algorithms) that are specified, designed and later implemented by the development team.

Common uses of a program:

  • Solve the accounting of a company.
  • Manipulate a robot.
  • Read and play audio, video, text files, etc.
  • Perform mathematical operations.

¿What is an Algorithm?

In formal terms, an algorithm is a fundamental logical unit. That is, a portion of reasoning. An algorithm allows you to solve a problem with a specific objective. We usually make an analogy between algorithms and cooking recipes. Thats it, if our problem to solve is 'Cooking a vanilla sponge cake', we can propose our recipe in an orderly way like an algorithm to solve the recipe or "the problem"


Unlike programs, they solve more general problems. Among these more general problems we can include:

  • Sort a messy data set.
  • Calculate the verification digit of the ID.
  • Encryption of a password.
  • Calculate the area of ​​a polygon.

¿What is a Flowchart?

A flowchart is simply a graphical representation of steps. It shows steps in sequential order and is widely used in presenting the flow of algorithms, workflow or processes. Typically, a flowchart shows the steps as boxes of various kinds, and their order by connecting them with arrows.

¿What is PseudoCode?

Pseudocode is an informal way of programming description that does not require any strict programming language syntax or underlying technology considerations. It is used for creating an outline or a rough draft of a program. Pseudocode summarizes a program’s flow, but excludes underlying details. System designers write pseudocode to ensure that programmers understand a software project's requirements and align code accordingly.

¿What is an Expression?

In computer science, an expression is a syntactic entity in a programming language that may be evaluated to determine its value. ... In simple settings, the resulting value is usually one of various primitive types, such as numerical, string, boolean, complex data, or other types of data.

Those are some of the elements that can conform an expression:

  • Literal values: 2 'hello' or 5.7 etc
  • Variables operators and procedure calls: 4 * 3 (This outputs the value 10 when evaluated by a computer)

Depending on the type of data Expressions can be clasified as Arithmetic, Relational or Logic expressions

Arithmetic Operetors

Arithmetic Operetors are the symbols that represent common arithmetic mathematical operations

  • + Plus: 5 + 7 = 12 (This outputs the sum of two integer numbers, two real numbers or a combination of both)
  • - Less: 12.0 - 5.5 = 6.5 (This outputs the subtraction of two integer numbers, two real numbers or a combination of both)
  • - Arithmetic negation: -45 (This outputs a change the sign of the operand from negative to positive or vice versa)
  • * Multiplication: 5 * 2.5 = 12.5 (This outputs a multiply of two integer numbers, two real numbers or a combination of both)
  • / Division: 10 / 3 = 3 (This outputs the subtraction of two integer numbers, two real numbers or a combination of both and the excess is discarded)
  • % Module: 10 % 3 = 1 (This outputs the excess of a divsion)
  • ^ Exponentiation: 5 ^ 2 = 25 (This outputs the elevation of the first operand to the power of the second operand)
Relational Operators

Relational Operators are the symbols that represents relations between two or more values

  • > Greater than: 8 > 15 = false
  • < Smaller than: 8 < 15=true
  • >= Greater or equal than: 8 >= 8 = true
  • <= Smaller or equal then: 8 <=-8=false
  • == Equals: "a" == "b" = false
  • <> Different: 10 <> 3 = true
Logic Operators

Logic Operators are the symbols that represents conditions between a number of values

  • && Conjuction: "a" == "b" && "c" < "d" (it is true if both statements are acomplished)
  • || Disjunction "a" == "b" || "c" < "d" (it is true if one of the statements is acomplished)
  • ! Negation: !("a"=="b") (it is true if the statement is not acomplished)
Complex Expressions

! Complex Expressions !

As in the operations used in arithmetic, in programming languages ​​the simple operations that we have seen so far can be combined into more complex operations and these are evaluated by performing the operations in a pre-established order. This order is specified by the precedence of the operators.

Operators with higher precedence are evaluated before operators with lower precedence. When the computer finds an expression with two operators that has the same level of precedence, it evaluates them from left to right.

As in basic mathematics, it is possible to alter the order of evaluation by using parentheses to group and order expressions. this happends in computer science beacause the parentheses is the expression with the smallest presedence and that means that any other expression inside it will be evalueted first.

Examples
  • Acceleration = (Final_Velocity - Initial-Velocity) / Time
  • Triangle_Area = Base_Lenght * Height_Length / 2
  • Obtained_Interest = Capital * Interest_Rate * Time / 100
Here is a list with the most common Presedence Order in programming
  • 1 Parentheses: ( )
  • 2 Arithmetic Negation -
  • 3 Exponentiation:: ^
  • 4 Multiplication, Division and Module: * / %
  • 5 Adittion and Subtraction
  • 6 Relational Operators: > <>= <==<>
  • 7 Logic Negation: !
  • 9 Conjuction and Disjunction: && ||
